# Galaga 技術規格書：02_遊戲機制詳解 (Gameplay Mechanics)

## 1. 玩家控制規格 (Player Controls)
*   **左右移動**: 數位輸入（左/右）。解析度需精確至單像素位移量。
*   **射擊 (Fire)**: 最大同畫面子彈數限兩枚。 ✅ (`GameApp.ts` line 753: `bullets.length < 2`)
    *   *技術限制*: 當畫面上已存在兩枚玩家子彈時，Fire 輸入應被忽略。 ✅
    *   *連發速率*: 每次射擊間隔 150ms (`PLAYER_FIRE_INTERVAL`)。 ✅

## 2. 牽引光束與雙機合體 (Tractor Beam & Dual Fighter)

### 2.1 牽引光束邏輯 (Tractor Beam Logic)
*   **觸發者**: 大首領 (Boss Galaga)。
*   **觸發條件**: 當 Boss 俯衝至畫面 `GAME_HEIGHT / 2` 以下高度且未受傷。 ✅
*   **光束判定**: 
    - 範圍：倒三角形區域，頂部寬度 16px，底部寬度 48px。
    - **視覺特效 (Animation)**: 青色 (`#00FFFF`) 半透明區域，帶有向下滾動的掃描線效果 (Scanning Lines)。光束邊緣具有呼吸燈式的頻率閃爍。
    - 持續時間：2.1 秒。 ✅ (`CAPTURE_DURATION = 2100`)
    - 若玩家進入區域，狀態切換至 `Spinning_Capture`，機體旋轉並被向上拉升。

### 2.2 雙機合體判定 (救援機制)
當大首領攜帶被俘戰機俯衝時，救援判定如下：

| 擊落對象 | 判定結果 | 效果描述 |
| :--- | :--- | :--- |
| **大首領 (俯衝中)** | **救援成功** | **動畫過程**: 被俘戰機失去動力旋轉下降，直到與現有机體 X 座標對齊，隨後產生一道閃光並「咔噠」一聲磁吸合體。 |
| **備用戰機 (由玩家誤擊)** | **救援失敗** | 戰機被摧毀，產生發紅的爆炸粒子（代表友軍誤傷）。 |
| **大首領 (編隊中)** | **救援失敗** | 被俘戰機轉變為敵對狀態（紅色），對玩家發起自殺攻擊。 |

### 2.3 雙機規格 (Dual Fighter Specs)
1.  **火力**: 每次攻擊同時發射兩顆平行子彈。
2.  **碰撞體積 (Hitbox)**: 
    - 單機寬度: 16px
    - 雙機寬度: 30px (重疊 2px 效果)
3.  **分數收益**: 擊殺獎勵維持單倍，但能更快清理編隊。

## 3. 敵機行為特徵 (Enemy Behaviors)
*   **變形 (Morphing)**: 部分敵機會在俯衝中分裂或變形為三隻小蜂（如 Stage 4+），此時其 Hitbox 將產生位移與分裂邏輯。 ✅ (`isTransforming` flag)
*   **子彈規避**: 高難度階段 (Rank > 10) 下，敵機將具有預測玩家位置並提前發彈的偏向。 ✅ (`Enemy.ts fire()` 方法，Rank > 10 啟用 Predictive Aiming)

---
> 🎮 **Logic Note**: 牽引光束的判定必須是矩形/三角形的遮罩偵測。玩家重心的 X 座標只要進入光束邊寬，即刻鎖定機體座標。
