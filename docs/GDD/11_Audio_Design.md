## 10. 音效設計 (Audio Design)

### 10.1 核心理念 (Core Philosophy)
H5 網頁遊戲在行動裝置上往往會在靜音下進行遊玩。因此，音效設計的首要目標是「強烈的反饋而無過度干擾」。
*   **環境音 (Ambience)**: 取消持續的背景音樂（除了進入關卡的開場旋律外），避免行動瀏覽器的音量過大。
*   **回饋音 (SFX)**: 針對擊中、陣亡、牽引光束三大事件給予清晰的電子合成音。針對雙發子彈，音效頻率要能有層次疊加（不能只是單純兩倍音量，需稍微錯開 0.05 秒）。

### 10.2 關鍵音效清單
所有音效使用 Web Audio API 即時合成，無外部音檔依賴。

| 事件 | 音效描述 | 合成方式 |
|---|---|---|
| 發射子彈 | 短促的 `piu` 聲，高音頻，低殘響。 | Square 波 800→300Hz, 0.1s 衰減 |
| 敵人爆炸 (Grunt) | 高頻白噪音爆炸音 | Sawtooth 150→10Hz, 0.3s 衰減 |
| 敵人爆炸 (Guard) | 中頻白噪音爆炸音 | Sawtooth 100→10Hz, 0.3s 衰減 |
| 敵人爆炸 (Boss) | 低頻深度爆炸音 | Sawtooth 60→10Hz, 0.3s 衰減 |
| 牽引光束命中 | 尖銳警報音，心跳般的脈衝節奏 | Square 波 1200/900Hz 交替 ×4, 每脈衝 0.12s |
| 合體成功 | 激昂兩階段和弦音 (Tada -- Dung!) | Triangle 波 C5→E5→G5→C6 琶音, 0.3s |
| 1UP 額外生命 | 上升琶音提示音 | Square 波 A4→C#5→E5→A5, 0.2s |
| 挑戰關卡通關 | 基礎三音和弦結算音 | Triangle 波 C5→E5→G5, 0.4s |
| 挑戰關卡 Perfect | 五音凱旋音效 | Triangle 波 C5→E5→G5→C6→E6, 0.4s |

### 10.3 雙發子彈音效處理
針對雙發子彈 (Dual Fighter)，主砲與副砲在 `fire()` 中分別呼叫 `playShootSound()`，因底層 Web Audio API 的時間精度差異 (~1ms)，自動形成微妙的時間錯開而非單純加倍音量。
