# Galaga 產出精確度檢查清單 (Production Checklist)

## 🛠️ 程式開發建議 (Programmer's Checklist)
- [ ] **時脈對齊**: 邏輯更新是否鎖定在 60Hz，並模擬 Z80 的指令週期延遲？
- [ ] **座標基準**: 所有的物件座標是否以 `288x224` 進行計算後再映射至螢幕？
- [ ] **渲染轉換**: 敵機在 Formation 下是否確實關閉了 Sprite 渲染，改用 Tile Map？
- [ ] **救援判定**: 是否已排除誤擊被俘機體導致的懲罰邏輯？
- [ ] **動態 Rank**: Rank 系統是否在 0-255 範圍內平滑運作，且無負溢出？

## 🎨 美術與視覺 (Artist's Checklist)
- [ ] **Sprite 規格**: 所有的敵機與子彈是否符合 `16x16` 的網格對齊？
- [ ] **雙幀動畫**: 敵機是否具備 2 幀的翅膀振動動畫？
- [ ] **色盤模擬**: 是否已套用 `BB GGG RRR` 的權重演算法處理色彩？
- [ ] **旋轉精靈**: 敵機俯衝時是否有對應的 8 個旋轉方向幀？

## 📊 數值與關卡 (Level Designer's Checklist)
- [ ] **五組規跡**: Stage 1 的五波進場路徑是否已通過硬編碼驗證？
- [ ] **分值倍率**: Boss 在攜帶 1 隻與 2 隻 Escort 俯衝時的得分 (400/800/1600) 是否正確？
- [ ] **挑戰關卡**: 全滅 40 隻敵人的 10,000 分大獎勵觸發是否正常？

---
> 🚀 **開發提示**: 本清單旨在確保最終作品具備「原始紅白機/街機」的精準神韻。
